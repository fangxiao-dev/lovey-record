# Module Management Page

## Purpose

The module management page is the canonical place for browsing and organizing the full module set.

## Responsibilities

- show the full module list or grouped sections
- preserve private/shared structure
- support management-state extensions later

## Must Preserve

- module entry remains clear and direct
- private and shared areas stay understandable
- selecting or entering a module still points to the same module instance

## Must Avoid

- replacing the dashboard as the default landing
- flattening all modules into a structureless list
- hiding ownership or shared-state context
