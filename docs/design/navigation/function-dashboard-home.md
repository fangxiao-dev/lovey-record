# Dashboard Home

## Purpose

The dashboard home is the first landing page after app entry.

Its job is to help users resume work fast.

## Responsibilities

- show recently used modules
- provide a clear path to module management
- keep the first screen light and scannable

## Must Preserve

- recent modules are shown before full-module browsing
- only used modules appear in the recent list
- users can jump to module management through a clear bridge action

## Must Avoid

- turning the page into a message feed
- listing all modules by default
- adding management-heavy controls to the first screen
