共享邀请流程

lovey-record · 设计资产 · 最后更新：2026-04-09

流程总览

创建人 → 参与人 → 数据库状态变化

创建人操作
参与人操作
系统 / 后端
前端页面
错误状态
① 创建人发起邀请
操作
模块管理页 点击「分享」
→
POST /createInviteToken
→
DB 写入 InviteTokentoken、过期时间(24h)、moduleInstanceId
触发
wx.shareAppMessagepath: pages/join/index?token=xxx
→
微信聊天中出现小程序卡片
② 参与人打开卡片
进入
pages/join/index
→
GET /validateInviteToken?token=xxx
校验
token 无效 / 已使用 / 已过期
→
展示错误页，引导回首页
token 有效
→
展示确认页模块名称 + 「只读」权限说明 + 「加入」按钮
③ 参与人确认加入
操作
点击「加入」
→
POST /acceptInvite { token }
DB
InviteToken.usedAt = now()
+
ModuleAccess 写入role: VIEWER, status: ACTIVE
+
ModuleInstance.sharingStatus = SHARED
跳转
进入共享模块（只读）
加入成功
④ 退出 / 移除
创建人
模块管理 → 移除参与人
→
POST /revokeModuleAccess已有接口
参与人
模块管理 → 退出共享
→
POST /leaveModule新接口（自己标记 REVOKED）
