## 页面范围

这次只做“模块首页整页”，不是全项目，也不是只做首屏。

页面必须覆盖以下区域：

- 状态卡
- 周期日历
- 图例
- 日期详情 / 原地编辑面板

## 不可改动的产品与交互约束

- 保留“状态卡 + 周期日历 + 原地编辑面板”的首页结构。
- 首页优先回答当前状态，不改成传统经期 App 那种整月日历主导页面。
- 月视图只是辅助浏览，不替代首页原地编辑。
- 单日点击后仍应在首页下方出现原地编辑面板，而不是跳去独立详情页。
- 长按滑动补录仍然是首页的主要批量编辑模型，不能被边缘化成次级入口。
- `period / prediction / today / special` 必须视觉上可区分，但层级克制。
- 共享语义始终是“同一模块实例共享访问”，不是复制一份数据给另一入口。
- 不要引入新的业务动作、复杂分析能力或额外页面跳转。

## 视觉方向

请沿着以下方向推进，不要偏离：

- 页面基底使用暖白、浅暖色、灰棕系。
- 卡片系统轻量、柔和、留白充足。
- 用更弱的阴影、更轻的边框和更统一的圆角建立视觉系统。
- 避免深色重卡片、厚描边、强 H5 感组件、情绪化“粉色经期工具”模板。
- 整体气质应更像克制、温和、微信原生感较强的工具产品。

## 微信原生小程序约束

- 设计必须适合微信原生小程序，而不是 Web 页面或 React App。
- 优先遵循微信官方设计指南和 WeUI 的组件语言。
- 不要默认输出 Web 导航、桌面布局或复杂浮层系统。
- 控件、层级、反馈和触控区域要符合小程序单手、短时使用场景。
