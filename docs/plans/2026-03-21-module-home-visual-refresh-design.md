# 模块首页视觉统一改版设计

**日期：** 2026-03-21

## 目标

在不改变月经模块核心交互和数据模型的前提下，统一首页的视觉语言，让状态卡、周期日历、图例和详情面板都靠拢同一套轻量、柔和、留白更充分的风格。

## 设计原则

1. 保留现有页面结构和交互入口，优先做视觉统一，不重做信息架构。
2. 日历区域成为首页主视觉，状态卡与编辑面板退居为同一视觉系统中的上下文卡片。
3. 使用暖白、浅粉、棕灰为主色，避免厚重深色块长期占据大面积区域。
4. 经期、预测、今天、特殊四类状态必须一眼可分，但层级要克制。
5. 周期窗和月视图共用同一套头部样式与日历语义，避免像两个页面拼接起来。

## 视觉方向

### 页面层

- 页面背景改为浅暖白渐变
- 主卡片统一更大圆角、更轻边框、更弱阴影
- 卡片之间通过留白区分，不靠硬描边堆层级

### 状态卡

- 保留入口标签、共享状态、模块名、主状态、次状态
- 取消目前偏“业务卡”的重边框感，改成更轻的奶白渐变卡片
- 主状态保留信息权重，但不再用过深的大块色压住页面

### 日历卡

- 顶部切换器改成更轻的 segmented control
- 周期窗头部改为：
  - 左右箭头式轻按钮
  - 中间两行标题：年份小字、月份主标题
  - 跨月或跨年时，中间显示区间标题
- 快捷跳转按钮改成更轻的小胶囊标签

### 日期格

- 普通日期默认不显示厚背景，只保留数字与充足留白
- 经期区间用浅珊瑚色胶囊连接带表达，日期数字落在更强的经期块上
- 预测窗口使用更淡的底色和虚线边界，不与经期争主层级
- 今天优先用小点和轻描边表达，不再使用沉重外框
- 特殊记录维持小圆点，但颜色更贴合新色板

### 图例与详情面板

- 图例放在浅暖底区域，缩小信息噪声
- 详情面板保留现有功能，但按钮、状态芯片、字段卡统一成同一套圆角与色板
- 编辑区的主次操作通过明暗和填充区分，不再依赖强描边

## 约束

- 不改变现有日历数据来源和状态判定规则
- 不增加新的业务动作
- 不把周历改成自然月日历
- 不删除首页原地编辑能力
